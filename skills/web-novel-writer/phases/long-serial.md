# 长篇续写阶段

将已有的短篇小说扩展为长篇网文，适配短剧拍摄（默认 100 集，约 200 章，20-25 万字）。

## 输入

- `{已有文本}`：前 N 章正文（通常为 10 章短篇的输出，作为卷一）
- `{故事设定}`：核心人物关系、金手指、世界观
- `{目标集数}`：短剧集数（默认 100 集，每集对应 2 章）
- `{题材方向}`：男频 / 女频及子类型

## 写前准备：加载参考资料

1. 读取 [../references/writing-rules.md](../references/writing-rules.md) 获取风格规则和禁用词。
2. 读取 [../references/emotion-model.md](../references/emotion-model.md) 获取情绪价值模型。
3. 读取 [../references/long-serial-arcs.md](../references/long-serial-arcs.md) 获取卷结构模板。
4. 如果是女频题材，读取 [../references/villain-dialogue-examples.md](../references/villain-dialogue-examples.md)。

## 核心架构：多卷制

将全书分为多个大卷（arc），每卷自成一个完整的"虐→转→爽"情绪闭环，卷间层层递进。

卷数和每卷章数根据目标总章数灵活调整。从 long-serial-arcs.md 中选择合适的卷结构模板，然后填入本作的具体设定。

### 默认六卷参考（200 章 / 100 集）

以下为参考分卷方式，实际使用时根据故事节奏调整每卷章数：

| 卷 | 参考章数 | 叙事功能 | 情绪振幅 |
|----|---------|---------|---------|
| 卷一 | ~10 章 | 起始冲突（通常为已有短篇内容） | E: -8 → +9 |
| 卷二 | ~35 章 | 余波与新威胁：旧冲突余震 + 新势力介入 | E: -6 → +7 |
| 卷三 | ~35 章 | 升级对抗：更大舞台、更强对手 | E: -7 → +8 |
| 卷四 | ~40 章 | 身世/真相翻转：颠覆认知的大秘密 | E: -9 → +8 |
| 卷五 | ~50 章 | 终极对决：幕后黑手现身、全面清算 | E: -10 → +9 |
| 卷六 | ~30 章 | 大结局：收官 + 番外彩蛋 | E: -3 → +10 |

**注意**：卷数不限于 6 卷，章数分配不限于上表。在阶段 A（卷大纲设计）中根据实际故事确定具体分卷。

### 卷主题设计原则

为每一卷确定主题时，遵循以下递进规律：

- **卷二**：旧冲突的余波尚未散尽，已有的胜利带来新的觊觎者
- **卷三**：舞台扩大（从家庭到行业/社会），反派量级升级
- **卷四**：信息翻转——关于主角/配角的核心身世或历史真相浮出水面
- **卷五**：所有暗线汇聚，终极 Boss 现身，主角面临最大危机
- **卷六**：清算所有旧账，给每条人物线一个交代

## 执行策略：逐卷推进

每一卷的执行分为 4 个阶段，必须按序执行。

### 阶段 A：卷大纲设计（串行）

为当前卷设计完整大纲，包含：

1. **卷概要**：200 字以内的卷情节概述
2. **新增角色表**：
   ```
   | 角色 | 身份 | 立场 | 首次出场章 | 核心动机 |
   ```
3. **逐章大纲**：每章包含——
   - 一句话情节摘要
   - 目标 E-Value（-10 到 +10）
   - 核心场景（地点、在场人物）
   - 章末钩子类型（悬念/反转/危机/情感/物品特写）
   - 短剧集归属（奇数章=集开头，偶数章=集结尾）
4. **伏笔计划**：
   ```
   | 伏笔 | 埋设章 | 预计回收章 | 回收方式 |
   ```
5. **本卷情绪曲线**：标注每 5 章的 E-Value 走向

将大纲呈现给用户，获得确认后再继续。

### 阶段 B：分批写作（5 章一批）

每批 5 章为一个写作单元。每批执行：

#### B.1 写前上下文准备

为本批构建上下文包：

```
上下文包 = {
  "前2章正文": "（完整文本，用于衔接语气和情节）",
  "角色状态表": "（当前所有角色的最新状态）",
  "伏笔清单": "（未回收的伏笔列表）",
  "本批大纲": "（本批5章的逐章大纲）",
  "情绪曲线": "（本批5章的目标E-Value）"
}
```

#### B.2 并行写作

并行启动 **2-3 个独立任务**，按以下分组：

- 如果 5 章内部有强因果依赖（如审判→判决），改为串行
- 否则按自然分组并行：如任务 1 写第 1-2 章，任务 2 写第 3-4 章，任务 3 写第 5 章

每个任务必须遵守：

**字数规范**：
- 每章 1000-1200 字（±5%）
- 对话占比 > 50%

**视角**：第一人称沉浸式（延续短篇风格）

**短剧适配**：
- 奇数章（集开头）：以动作/事件开场，快速切入冲突
- 偶数章（集结尾）：必须以强悬念/反转收尾，制造"下一集"欲望
- 每 2 章（一集）内部节奏：冲突建立 → 矛盾升级 → 悬念爆发

**章末钩子**（遵循 short-novel.md 中的禁止模式规则）：
- 奇数章结尾：中等悬念，引导读者继续本集
- 偶数章结尾：强悬念/反转/危机，制造集间断点

**角色一致性**：
- 已退场角色不能无理由重新出现
- 关系状态必须与角色状态表一致
- 新角色首次出场必须有身份交代

#### B.3 批次校验

每批 5 章写完后执行：

```bash
novel-cli validate <输出文件> --mode long
novel-cli scan <输出文件>
```

如发现禁用词或机械断句，先修复问题章节；校验通过后，更新以下状态：

**角色状态表**：
```
【角色状态·第XX章后】
| 角色 | 状态 | 所在地 | 关键关系变化 |
```

**伏笔清单更新**：
```
【伏笔追踪·第XX章后】
| 伏笔 | 状态 | 埋设章 | 回收章 | 备注 |
```

### 阶段 C：卷末审稿（串行）

一卷全部写完后：

1. 通读全卷，检查：
   - 角色行为一致性
   - 时间线连贯性
   - 伏笔回收率（本卷主要伏笔必须 80% 以上回收）
   - 情绪曲线是否符合预设
2. 运行全卷校验：
   ```bash
   novel-cli validate <全卷文件> --mode long --from <卷起始章> --to <卷结束章>
   ```
3. 对问题章节进行润色修正
4. 生成本卷摘要（每章一句话），作为下一卷的上下文输入

### 阶段 D：卷间衔接

当前卷完成后，进入下一卷前：

1. 将上一卷末尾的角色状态表、伏笔清单、卷摘要打包为下一卷的输入
2. 回到阶段 A，为下一卷设计大纲
3. 询问用户是否需要调整后续卷的方向

## 情绪节奏设计

### 卷内节奏：微波形

每卷内部，每 5-7 章构成一个"小虐-小爽"周期：

```
章 1-3: 新冲突引入，E-Value 下降
章 4-5: 小反转，E-Value 回升
章 6-7: 更大冲突，E-Value 再降（更低）
章 8-10: 阶段性胜利，E-Value 上升
...循环递进...
卷末 3 章: 本卷大高潮，E-Value 达到卷内峰值
```

### 卷间节奏：大波形

每卷结尾的高点，到下一卷开头必须回落——新威胁出现拉低 E-Value，然后再爬升。振幅逐卷递增：

```
卷一 → 卷二：回落幅度小（新势力试探）
卷二 → 卷三：回落幅度中（更大阴谋浮现）
卷三 → 卷四：回落幅度大（身世/真相打击）
卷四 → 卷五：回落幅度最大（终极危机）
卷五 → 卷六：不再大幅回落（收官模式，保持 E > +5）
```

### 短剧集节奏

每 2 章 = 1 集。每集内部节奏：

| 位置 | 功能 | 占比 |
|------|------|------|
| 集开头（奇数章前 30%） | 前情回顾 + 新冲突切入 | 600-700 字 |
| 集中段（奇数章后 70% + 偶数章前 50%） | 矛盾升级 + 情节推进 | 1200-1400 字 |
| 集结尾（偶数章后 50%） | 高潮 + 悬念钩子 | 500-600 字 |

## 新反派引入规则

每卷至少引入 1 个有分量的新对手，维持长篇张力。反派量级逐卷递增：

| 卷 | 新反派量级 | 与主角关系类型 | 威胁等级 |
|----|----------|-------------|---------|
| 卷二 | 小 Boss | 间接威胁（旁系势力、商业对手） | ★★☆ |
| 卷三 | 中 Boss | 直接对抗（行业/权力层面） | ★★★ |
| 卷四 | 大 Boss 线索 | 亲情/血缘纠葛 | ★★★★ |
| 卷五 | 终极 Boss | 全面碾压 + 信息优势 | ★★★★★ |
| 卷六 | 无新反派 | 清算旧账，给所有人物线收尾 | — |

设计反派时遵循：
- 每个反派有独立的动机，不是纯粹的"坏"
- 反派的行为逻辑必须自洽
- 低等级反派可以被高等级反派利用/吞噬，形成食物链

## 长篇特有的防崩措施

1. **角色上限**：全书有名字的角色不超过 30 个，每卷新增不超过 5 个
2. **场景复用**：70% 的场景应该在已建立的地点发生，新场景每卷不超过 3 个
3. **时间线锚点**：每 10 章必须有一个明确的时间标记（季节、节日、怀孕周数等）
4. **前后呼应**：每卷至少回调 2 次卷一的经典场景/台词（变奏式呼应）
5. **语气衰减防控**：每 20 章做一次风格校准——对比第 1 章的句式、节奏，矫正偏移
