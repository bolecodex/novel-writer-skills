# 黄金三章阶段

写出网文的前3章——决定读者去留的关键段落。

## 输入

- 故事设定（来自脑洞阶段或用户直接提供）
- 题材方向（男频/女频及子类型）

## 写前准备：加载规则

写作前，读取 [../references/writing-rules.md](../references/writing-rules.md) 加载完整的写作规则、禁用词表和风格约束。

## 执行：串行（3章之间有情节依赖）

不要并行写章节。每一章都建立在上一章的基础上。

### 预备步骤：内部架构设计

在写任何正文之前，先在内部确定：
- 三章的情节弧线、冲突点和反转时刻
- 主角的金手指是什么、在哪里使用
- 每章具体的打脸/反转情节
- 第1章和第2章的章末钩子

### 每章结构（每章2200-2600字，±5%）

每章必须按顺序包含以下部分：

**1. 开篇抓人（300-500字）**
- 开局即冲突，给读者一个立刻关心的事件
- 用低成本快速曝光主角优势原型
- 装载1-2个悬而未决的信息点（震惊、疑问、可预见爽点）

**2. 人设亮点与爽点爆发（1000-1500字）**
- 主角展现金手指，解决燃眉之急
- 爽点需可视化：反转、碾压、智斗、资源获取、身份高亮
- 至少埋一个情绪反差（被看低→亮身份；陷困境→瞬逆袭）

**3. 情绪拉扯与埋线（600-1000字）**
- 引入更大冲突或隐藏威胁
- 暴露当前优势的限制或代价，留出脆弱点增加张力
- 抛出下一章必须见分晓的问题

**4. 章节结尾**
- 必须以角色的对话或动作结束，拒绝总结和未来展望
- 最后1-2句必须引导读者立即翻页（急迫感、悬念、危机）

### 写作规则（强制执行）

- **视角**：严格的第三人称有限视角。读者信息=主角信息。
- **句式**：70%短/中句（8-15字），30%长句（20-35字）。禁止比喻。禁止修饰性副词。
- **对话**：功能性——推动情节、传递信息、体现群体态度。简短直接。
- **金手指**：每章出现4-7次，驱动情节解决。
- **信息传递**：通过动作和对话传递，不要用大段旁白铺垫。

### 反套路策略

每章必须有：
- 1个主要反套路转折（身份反转、规则反转、动机反转）
- 2-3个细节性意外，增加阅读新鲜感

### 数据化指标

| 指标 | 目标值 |
|------|--------|
| 章节字数 | 2200-2600字 ±5% |
| 心理描写占比 | 15-25% |
| 金手指出现率 | 每章4-7次 |
| 首个爽点时机 | 前10%字数内 |

## 写后校验

三章全部写完后执行：

```bash
novel-cli count <输出文件>
novel-cli scan <输出文件>
novel-cli validate <输出文件> --mode golden
```

如果任何章节不在2200-2600字范围内，修改。如果发现禁用词或机械断句，修复后再交付。
