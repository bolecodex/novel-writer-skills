# 脑洞生成阶段

将用户的原始灵感转化为3个完整的、可直接开书的故事方案。

## 输入

- `{灵感}`：用户提供的火花——关键词、主题、场景或模糊想法
- `{方向}`：男频 或 女频

## 执行：三路并行

并行启动 **3个独立任务**，每个任务独立使用不同的公式生成一个完整故事方案。

### 每个任务的指令

1. 读取 [../references/formulas.md](../references/formulas.md) 加载公式库。
2. 选择一个最匹配灵感的公式。3个任务必须使用**不同的**公式。按索引分配：任务1用第一个匹配公式，任务2用第二个，任务3用第三个。如果匹配的公式不足3个，从番茄爆款题材中补充。
3. 将用户的灵感代入公式的变量槽位。
4. 按以下结构生成完整方案：

```
## [吸睛标题]

**标签**：【关键词1+关键词2+关键词3+关键词4+关键词5】

### 核心构成要素
1. 主角：...
2. 金手指：...
3. 核心玩法：...
4. 世界观：...

### 关键爽点
- 爽点1
- 爽点2
- 爽点3（至少3个，第一个必须在10章内可回收）

### 爆款文案
（12-18行，一句一行，番茄小说风格）

### 前三章一句话剧情
第一章：...
第二章：...
第三章：...

### 爽点循环
（例：职业认知 → 现象观察 → 逻辑解构 → 跨界应用 → 效果颠覆 → 众人震惊 → 获得资源 → 深化认知）
```

## 价值优先级（按序）

1. **创新性** — 组合要新颖，让人眼前一亮
2. **爆款潜力** — 贴合当前网文市场流行趋势
3. **逻辑自洽** — 设定必须内在逻辑通顺
4. **可执行性** — 作者拿到手就能直接开写第一章

## 题材规避（除非用户主动要求）

避免：科技、科幻、赛博朋克、AI、熵增、法则、退婚

## 结果汇总

3个任务全部返回后，将3个方案一起呈现给用户。让用户选择最喜欢的（或请求修改）。选中的方案成为黄金三章或10章短篇的输入。
